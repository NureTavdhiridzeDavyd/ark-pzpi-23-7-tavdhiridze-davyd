1. РОЗГОРТАННЯ СИСТЕМИ
     Серверну частину програмної системи HiveTrack було розгорнуто на хмарній платформі Render як Web Service із підключенням до GitHub-репозиторію. Для коректної збірки проєкту було вказано кореневу директорію hivetrack, після чого Render автоматично виконав встановлення залежностей (npm install) та запуск застосунку (npm start). Після завершення деплою сервіс отримав публічну URL-адресу, а працездатність бекенду підтверджено через перегляд логів розгортання та успішний запит до ендпоінта перевірки стану /api/health

Рисунок 5.1 – Розгорнута Backend-частина
     Після розгортання бази даних MySQL у Railway виконано підключення серверної частини HiveTrack, розгорнутої на Render, до хмарної БД через змінні середовища. У налаштуваннях Render для сервісу були задані параметри підключення відповідно до даних Railway, після чого сервіс було перезапущено. Успішність з’єднання підтверджено відсутністю помилок у логах Render та коректною роботою API після запуску.

Рисунок 5.2 – Розгорнута База даних
     IoT-модуль реалізовано на базі ESP32 з підключенням до мережі Wi-Fi Wokwi. Після старту пристрій ініціалізує датчики та інтерфейс відображення (OLED SSD1306), під’єднується до заданої Wi-Fi мережі та формує телеметрію з вхідних значень (швидкість, інтенсивність гальмування, відхилення від смуги та температура DS18B20). Зібрані дані періодично відображаються на екрані й паралельно надсилаються HTTP-запитом на серверний API (у тестовому режимі використовується постман-echo, у реальному — ендпоінт бекенду HiveTrack). Таким чином, симуляція у Wokwi підтверджує коректність підключення компонентів, обробку показників і передачу даних у мережу для подальшого збереження та аналізу в системі.
Посилання на проєкт: https://wokwi.com/projects/449961177807696897
